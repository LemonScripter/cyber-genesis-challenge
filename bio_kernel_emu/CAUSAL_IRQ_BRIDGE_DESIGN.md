# [HU] Kauzális IRQ Híd (Causal IRQ Bridge)
**Szerepkör:** Kapocs a fizikai világ (felhasználói interakció) és a digitális logika között. Ez a komponens felel a „Szándék” (Intent) hitelesítéséért.

## 1. Az események osztályozása
A híd kétféle eseményt különböztet meg a böngészőben:
- **Hardveres Megszakítás (Trusted IRQ):** Valódi fizikai események (pl. `mousedown`, `touchstart`, `keydown`), amelyeket a böngésző `isTrusted` flag-je hitelesít.
- **Szoftveres Szimuláció (Untrusted):** Scriptek által generált események (pl. `element.click()`). Ezeket a BioOS automatikusan elutasítja.

## 2. A Kauzális Ablak (Causality Window)
A „Szándék” nem örökérvényű. Egy IRQ generálása után megnyílik egy szűk időablak (pl. 200 ms):
- Ha a parancs ezen az ablakon belül érkezik, az okozati lánc **ÉRVÉNYES**.
- Ha az ablak bezárul, az okozati lánc **MEGSZAKAD**, és az IRQ érvénytelenné válik.
- *Ez megakadályozza, hogy egy vírus „eltárolja” a felhasználó korábbi kattintását későbbi felhasználásra.*

## 3. IRQ Token Generálás
Amikor egy hiteles IRQ történik, a híd egy egyedi **Kauzális Tokent** generál a State Manager számára. Ez a token tartalmazza:
- Az interakció típusát (pl. BUTTON_SAVE).
- A pontos időbélyeget.
- A fizikai koordinátákat (hogy a szándékot térben is lehessen korlátozni).

# [EN] Causal IRQ Bridge
**Role:** The link between the physical world (user interaction) and digital logic. This component is responsible for authenticating "Intent".

## 1. Classification of Events
The bridge distinguishes between two types of events in the browser:
- **Hardware Interrupt (Trusted IRQ):** Genuine physical events (e.g., `mousedown`, `touchstart`, `keydown`) authenticated by the browser's `isTrusted` flag.
- **Software Simulation (Untrusted):** Events generated by scripts (e.g., `element.click()`). BioOS automatically rejects these.

## 2. The Causality Window
"Intent" is not permanent. After an IRQ is generated, a narrow time window opens (e.g., 200 ms):
- If the command arrives within this window, the causal chain is **VALID**.
- If the window closes, the causal chain is **BROKEN**, and the IRQ becomes invalid.
- *This prevents malware from "storing" a user's previous click for later use.*

## 3. IRQ Token Generation
When a trusted IRQ occurs, the bridge generates a unique **Causal Token** for the State Manager. This token contains:
- The interaction type (e.g., BUTTON_SAVE).
- Precise timestamp.
- Physical coordinates (to restrict intent spatially).
